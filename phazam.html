SUMMARY

Have you ever wanted to test your ability to look at a photo and identify how it was shot? With Phazam, you can.

Phazam is a mini web application that pulls images directly from Flickr and presents it in a unique way - emphasizing the technical details behind a photo. Hover over an image to display its EXIF data: focal length, exposure, aperture, and ISO speed. Use the gallery of photos to warm up and browse the photos. Test yourself in the challenge section to see if you're as accurate as you think you are.

Phazam retrieves 40-50 images from Flickr's 'Interesting Photos of the Day' API every day.

MOTIVATION

With photography becoming increasingly popular, we look to our favorite artists and photos for motivation to continue improving our skills. By looking at the settings used to create a shot, we as photographers can gain insight as to how we can go about re-creating our favorite images. By evaluating focal length, we can assess how far the photographer was from their subject. Shutter speed can provide some insight on whether the subject was moving fast or if the photo was done with a long exposure. Aperture influences the depth of field in the photo. We can learn to adjust aperture based on what we want in focus. ISO speed can tell us how much light was available and tells us how sensitive the sensor needed to be. The combination of these settings help photographers understand the way they work together to create a fantastic photo. My hope in creating this web application is to help others develop their photographic eye and continue creating fantastic photos.

TECHNICAL DETAILS

Phazam leverages the FlickRaw gem to simplify the Flickr API. It firsts retrieves a set of images from Flickr's 'Interesting Photos of the Day' API. It then calls a separate API for each of the images in order to retrieve its associated data (EXIF attributes). These images are then stored into the database to reduce redundant API calls and improve webpage loading times. Each challenge is stored in its own object. The object contains information on the user's guesses as well as a link to the image object that the challenge is for.

CHALLENGES & SOLUTIONS

  • I struggled with understanding how to parse out the JSON objects that the Flickr API returned. The objects were massive combinations of arrays and hashes and I was trying to find a simple way to handle all the incoming data and parse out only what I needed
    • Solution: I eventually learned about JSON.parse which does a lot of the initial lifting for you. But I ended up using a combination of Ruby and customized methods to dig down into the elements that I needed
    • My solution is more precise in grabbing what I personally need at this time but it does have limitations. I would need to re-work my algorithm to accommodate other attributes if needed in the future
  • Using the API led to very long loading times (approximately 30 seconds to fully load 40-50 images). There was a API call to gather all the images and for each image, there was an API call to gather its respective data.
    • I improved the user experience by reducing the number of times the API is called. I did this by implementing a database check for images created today. If they exist, then that indicates that the API was already used today and there is no need to call the API again. Instead, we will just load up the images from the database. This reduced the load times experienced by users from 30 seconds to less than a second!
  • Initial API call still takes about 30 seconds to initiate per day
    • A scheduled task was implemented to seed the database every morning.
    • A concern was that this will flood my database
    • Another scheduled task was implemented to purge the database of old images (> 2 weeks old)
  • HAML
    • On this project, I challenged myself to fully use HAML and learned the syntax and structure of the pre-processor
